# Reproducible script templates

At the end of every run, emit a single runnable script that takes the raw input file(s) → produces the final cleaned dataset. The user picks Python or R during Discovery (Python is the recommended default).

## Principles

- **Single file, top to bottom.** The script reads raw, applies every transformation in order, writes the final dataset. No imports of other generated files.
- **All decisions are baked in.** The user already made the choices during the interactive run; the script records them as constants at the top.
- **Use the canonical libraries.** Python → pandas. R → tidyverse.
- **Comment every transformation block** with one sentence: what's happening and why.
- **Emit the same intermediate artifacts** (audit CSV, ledger CSVs) so re-running produces identical bundle.
- **Idempotent.** Re-running on the same raw input produces the same cleaned output bit-for-bit.

## What to include vs not include

**Include in the script:**
- All row filters with their thresholds as named constants at the top
- All Likert / categorical recodings with explicit maps
- Multi-select option lists (the canonical labels)
- The numeric-text-parsing function inlined
- Strategic binary rules
- Output file paths

**Don't include in the script:**
- Interactive elicitation calls (decisions are already baked in)
- The data profile generation (analyst can re-run that with a data-exploration workflow if needed)
- The narrative process documentation generator

## Verification

After generating the script, validate by running it against the raw input and comparing the output to the dataset produced by the interactive run. If they match (same row count, same column names, same values for a sample of cells), declare the script verified. If not, fix the script before declaring the run complete.
